# 11_Dashboard_Orchestration_Engine

Version 1.0

---

# Purpose

The Dashboard Orchestration Engine (DOE) is responsible for converting business understanding into an interactive dashboard.

It decides:

What should be displayed

How it should be displayed

Where it should be displayed

When it should be updated

The frontend should only render the dashboard configuration generated by this engine.

It should never decide which charts to create.

---

# Responsibilities

The DOE is responsible for:

Generate initial dashboard

Recommend visualizations

Arrange dashboard layout

Manage widgets

Prevent duplicate visualizations

Handle AI dashboard actions

Manage filters

Update dashboard state

Generate Plotly configurations

Produce dashboard JSON

# Default Dashboard Composition

Every dashboard should include:

1. Executive KPI Cards

2. Trend Chart

3. Category Comparison

4. Regional Comparison (if available)

5. Distribution Chart

6. Top N Table

7. AI Summary Panel

8. Chat Panel

The dashboard should never be empty.

---

# KPI Selection Rules

Automatically prioritize:

Revenue

Profit

Sales

Orders

Customers

Growth

Margin

Quantity

Display a maximum of four KPI cards by default.

---

# Chart Recommendation Rules

Choose chart types based on metadata.

Examples:

Date + Metric

→ Line Chart

Category + Metric

→ Bar Chart

Few Categories + Metric

→ Pie Chart

Many Categories + Metric

→ Horizontal Bar Chart

Two Metrics

→ Scatter Plot

Distribution

→ Histogram

Hierarchy

→ Treemap

Never choose charts that are unsuitable for the data.

---

# Widget Layout Rules

Large KPI cards at the top.

Trend chart spans full width.

Comparison charts below.

Tables near the bottom.

Chat panel fixed on the right (desktop).

Responsive stacking on mobile.

Avoid empty spaces.

Maintain visual balance.

---

# Widget Model

Each widget contains:

Widget ID

Title

---

# Filter Generation

Automatically generate filters for:

Date

Category

Region

Product

Customer

Location

Do not create filters for:

Revenue

Profit

Quantity

Identifiers

---

# AI Dashboard Actions

The engine must support:

Create Chart

Delete Chart

Replace Chart

Resize Widget

Move Widget

Add KPI

Remove KPI

Apply Filter

Clear Filter

Sort Table

Reset Dashboard

Every action updates the Dashboard JSON.

---

# Duplicate Prevention

Do not create duplicate widgets.

Example:

If "Revenue by Month" already exists and the user requests it again,

focus the existing widget instead of creating another.

---

# Dashboard Personalization

Remember user preferences during the session.

Examples:

Preferred chart type

Collapsed panels

Widget positions

Applied filters

Theme

These preferences are stored in client state only.

---

# Plotly Configuration

Generate Plotly configuration objects for every visualization.

The frontend should pass these objects directly to Plotly without additional transformation.

---

# Empty States

No suitable metric

Show a friendly message.

No date column

Hide trend chart.

No category column

Hide comparison chart.

No rows after filtering

Display "No matching data found."

---

# Dashboard Update Rules

Updates should be incremental.

Do not regenerate the entire dashboard for every AI request.

Modify only the affected widgets.

Preserve:

Scroll position

Filters

Widget positions

User customizations

---

# Success Criteria

The Dashboard Orchestration Engine should produce a polished, business-friendly dashboard with:

Meaningful KPIs

Appropriate visualizations

Logical layout

Interactive filters

Downloadable widgets

Incremental updates

No duplicate charts

Minimal user configuration

The generated dashboard should immediately answer:

How is my business performing?

What changed?

Where should I focus?

without requiring the user to build anything manually.
